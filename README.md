# HashLib — анализ файлов по хэш-суммам

## Описание проекта

Проект представляет собой консольный программный скрипт, реализующий механизм обнаружения вредоносных файлов на основе сравнения их хэш-сумм с локальной и удалённой базами данных.

Система использует сигнатурный подход: каждый файл представляется в виде уникальной хэш-суммы (SHA-256), которая сравнивается с известными сигнатурами вредоносных объектов.

---

## Назначение

* анализ файлов на наличие вредоносного кода
* демонстрация сигнатурного метода обнаружения
* учебное применение в области информационной безопасности
* базовый инструмент проверки файлов

---

## Архитектура проекта

Проект построен по модульному принципу и состоит из следующих компонентов:

### main.py

Центральный управляющий модуль.
Организует полный цикл работы программы: ввод данных, обработку файлов, взаимодействие с базами и вывод результатов.

---

### scanner.py

Модуль сканирования файловой системы.
Выполняет рекурсивный обход директории и формирует список файлов для анализа.

---

### hasher.py

Модуль вычисления хэш-сумм.
Использует алгоритм SHA-256 для получения уникального идентификатора файла.

---

### database.py

Модуль работы с локальной базой данных SQLite.
Обеспечивает:

* создание базы
* хранение хэшей
* поиск совпадений

---

### api_client.py

Модуль взаимодействия с внешним сервисом VirusTotal.
Отправляет HTTP-запросы и обрабатывает ответы API.

---

### db_tool.py (вспомогательный)

Утилита для управления базой:

* добавление хэшей
* просмотр базы

---

### import_json_to_db.py (вспомогательный)

Скрипт для массового импорта хэшей из JSON-файла.

---

## Принцип работы системы

Алгоритм работы программы:

1. Пользователь указывает путь к директории
2. Выполняется сканирование файлов
3. Для каждого файла вычисляется SHA-256 хэш
4. Производится проверка в локальной базе данных
5. При отсутствии совпадения выполняется запрос к удалённой базе
6. Определяется статус файла
7. Результаты выводятся пользователю
8. Формируется итоговая статистика

---

## Алгоритм (схема)

```
ФАЙЛ → SHA-256 → ЛОКАЛЬНАЯ БД → (если нет) → VirusTotal → РЕЗУЛЬТАТ
```

---

## Работа с хэшами

В проекте используется алгоритм:

* SHA-256

Хэш-функция преобразует содержимое файла в строку фиксированной длины (64 символа), которая используется как уникальный идентификатор.

Сравнение выполняется по принципу:

* полное совпадение хэш-сумм

---

## Работа с локальной базой данных

Используется SQLite.

Структура таблицы:

```
hashes(
    hash TEXT PRIMARY KEY
)
```

Особенности:

* хранение хэшей вредоносных файлов
* уникальность записей
* быстрый поиск по индексу

---

## Работа с удалённой базой

Используется API сервиса VirusTotal.

Механизм:

* отправка SHA-256 хэша
* получение JSON-ответа
* анализ статистики

Основные параметры ответа:

* malicious
* suspicious
* undetected

---

## Классификация файлов

В системе используются статусы:

* MALICIOUS — вредоносный файл
* SAFE — безопасный файл
* UNKNOWN — отсутствует в базе

Источник:

* LOCAL — локальная база
* ONLINE — удалённая база
